## ğŸ’¡ Key SQL Concepts and Functions in Schema Creation

The `CornerShop_Foreigh.sql` file primarily focuses on schema definition. Here are the core SQL concepts utilized:

### 1. Schema Definition and Data Types ğŸ› ï¸

| SQL Command/Concept | Description | Function in CornerShop Schema |
| :--- | :--- | :--- |
| **`CREATE SCHEMA`** | Command used to create a new database container. | Creates the **`cornershop`** database. |
| **`CREATE TABLE IF NOT EXISTS`** | Defines a new table structure only if the table does not already exist. | Defines all five tables with columns and constraints. |
| **`INT NOT NULL`** | Data type for integers that cannot contain null values. | Used for all primary keys (e.g., `Customer_ID`) and required numeric fields. |
| **`VARCHAR(45)`** | Data type for variable-length strings, limited to 45 characters. | Used for names, addresses, and email fields. |

### 2. Integrity Constraints & Rules ğŸ”’

| Constraint/Rule | Description | Purpose in CornerShop Schema |
| :--- | :--- | :--- |
| **`PRIMARY KEY`** | A constraint that **uniquely identifies** each record in a table. | Guarantees unique identification for customers, products, etc. |
| **`FOREIGN KEY`** | A constraint linking a column in one table (child) to the primary key of another table (parent). | Enforces referential integrity between tables. |
| **`ON DELETE NO ACTION`** | **Referential Integrity Rule:** Rejects deletion of a parent record if dependent child records exist. | Prevents accidental deletion of customers or products that have existing orders. |
| **`ON UPDATE CASCADE`** | **Referential Integrity Rule:** If the primary key in the parent table is updated, the foreign key in the child table is automatically updated to match. | Ensures consistency if ID numbers need to be changed. |
